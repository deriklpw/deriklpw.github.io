Android事件分发机制

一、关于事件分发的几个问题

事件分发机制有点复杂，以源码为依据，从自己理解的角度出发，稍微整理一下整套机制中，需要理清的几个问题，最后再附一个简单的例子。

1. 为什么要进行事件分发？

用户在Android系统屏幕上进行操作后，会有相应的事件产生。当产生事件的区域，有多个组件可以响应时，Android系统需要事件分发机制，来决定该事件传递给哪一个组件进行处理。
2. 什么是事件分发？

事件分发，是Android系统对用户行为产生的事件，进行传递处理的过程。
3. 事件分发指的是什么事件？

MotionEvent事件，具体可以是：ACTION_DOWN, ACTION_UP, ACTION_MOVE, ACTION_CANCEL等。
4. 如何进行事件分发？

Android展现出来的界面，是一层一层的View叠加而成。 事件分发采用了责任链式的设计模式，事件层层传递，从上往下，再从下往上，寻找可以处理的组件，未找到，则将该次事件丢弃。

二、事件分发详解

事件分发的主体主要是ViewGroup和View，虽然ViewGroup也是继承自View，但是在事件的处理上ViewGroup和View所需要考虑的因素不同，处理过程有所不同，因为ViewGroup除了要考虑自己，还需要考虑其中的各个子View。至于Activity，本质上也是ViewGroup来着，只是没有拦截方法而已（事件从它而来，拦截没有意义），理解了ViewGroup事件分发，Activity差不多也通了。所以，事件分发可以分为两类：

  • ViewGroup的事件分发

  • View的事件分发

1、ViewGroup事件分发：

ViewGroup事件分发主要涉及到以下三个方法：

  • public boolean dispatchTouchEvent(MotionEvent ev)。
  • public boolean onInterceptTouchEvent(MotionEvent ev)。
  • public boolean onTouchEvent(MotionEvent ev)。
    其中的核心方法是dispatchTouchEvent()，所以从这个方法开始分析，至于后面两个方法，它们会在dispatchTouchEvent()的执行过程中调用。开始贴源码了，源码较多，抓住关键点进行分析与理解。
