Java 8的四个重要新特性总结

Java 8

Java 8的新特性有很多，只对代表性的四个重要特性做一次总结，写成博客，方便以后回顾。

Java8的四个重要新特性:

  1. Lambda
  2. 方法引用
  3. 默认方法
  4. Stream

1. Lambda

Lambda表达式：Lambda可以让函数当做一个方法的参数进行传递，并且让代码变得更加简洁。lambda表达式省去了匿名类的麻烦，但Lambda只能给包含一个方法的接口定义，且lambda的入参和返回值必须和接口中的方法一致。
语法：
(type param1, type param2…) -> {Statements}
重要特性：

  1. 参数类型可选：编译器可以自动识别参数的类型。
  2. 参数圆括号可选：只有一个参数时，可省略圆括号。
  3. 语句块大括号可选：只有一个语句时，可省略大括号。
  4. 返回值可选：只有一个语句时，可省略返回值。

LinearLayout android:layout_weight 属性用法总结

LinearLayout是很常用的一种布局，当考虑使用LinearLayout的时候，为了在布局时，达到较好的屏幕适配效果，可以选择使用android:layout_weight属性。那么，当为布局中的每个组件指定了权重之后，我们的android系统是如何给组件计算大小的呢？一起看看吧！

各组件大小的计算方式

实际大小=指定大小+剩余大小*权重比例，
剩余大小=屏幕大小-（所有组件大小的和），
所以最后为：实际大小=指定大小+（屏幕大小-（所有组件大小的和））*权重比例。

例如:

在LinearLayout中放置三个按钮，分别为Button1，Button2，Button3，指定android:layout_height="wrap_content"不变，每个Button的android:layout_weight分别为1，2，3。

依据android:layout_width属性的设置，分三种情况讨论android:layout_weight属性，L为屏幕大小（以宽为例）：

  1. android:layout_width="0dp"时：

    组件大小=0+（L-（0+0+0））*权重比例，组件大小和权重成正比。

    分别计算下各Button占的宽度：
    Button1的大小=0+(L-(0+0+0))*(1/6)，为1/6L；
    Button2的大小=0+(L-(0+0+0))*(2/6)，为2/6L；
    Button3的大小=0+(L-(0+0+0))*(3/6)，为3/6L。
    所以Button1:Button2:Button3 = 1:2:3。

  2. android:layout_width="wrap_content"时：

    组件大小=所需大小+(L-各组件所需大小和)*权重比例，剩余大小即(L-各组件所需大小和)为正，则为正比，否则为反比。

    该情况和组件内容占用大小有关，所以布局上并不是很明显的比例关系，因为比例关系仅指剩余大小的比例：

    1. 剩余大小为正，即屏幕空间足够，所占和权重成正比。
    2. 剩余大小为负，即屏幕空间不足，所占和权重成反比，下面的图可能有点难看，不过意思到了。

  3. android:layout_width="match_parent"时：

    组件大小=L+（L-（L+L+L））*权重比例，组件大小和权重成反比，如果前面组件已经占满屏幕，则剩余组件在屏之外位置，不可见。

    继续计算各Button占的宽度：
    每个Button的layout_width指定为match_parent，即表示每个Button指定大小为父组件宽度，即L。
    Button1的大小=L+(L-(L+L+L))*(1/6)，为4/6L；
    Button2的大小=L+(L-(L+L+L))*(2/6)，为2/6L；
    Button3的大小=L+L(L-(L+L+L))*(3/6)，为0。
    所以Button1:Button2=2:1，Button3不可见。

总结

LinearLayout中各组件大小的计算方式：实际大小=指定大小+剩余大小*权重比例，按照这个计算方式去理解和使用，android:layout_weight属性用起来很简单。
